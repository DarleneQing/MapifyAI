# Backend Synthesis Frontend Handoff

Date: 2026-03-16
Status: Short handoff note for frontend integration planning

## What Backend Already Provides

Current graph order:

1. intent_parser
2. crawling_search
3. transit_calculator
4. evaluation_agent (parallel)
5. review_agent (parallel)
6. orchestrator_agent
7. output_ranking
8. synthesis_agent
9. END

Current backend response contract:

- Non-stream: `{ request, results, agent_reply }`
- Stream final event: `{ type: "result", request, results, agent_reply }`

`agent_reply` is generated by backend synthesis and also has a template fallback when LLM output is empty.

## Round 1 Backend Structure

Helper modules introduced:

- `backend/app/agents/synthesis_prompts.py`
  - Prompt assembly for synthesis (system/user messages)
- `backend/app/services/synthesis_context.py`
  - Context item build + top-context selection

Config knobs (defaults):

- `MAX_PLACES_IN_CONTEXT = 3`
- `SYNTHESIS_BUDGET_TOKENS = 2000`
- `ENABLE_REVIEW_SIGNALS = True`
- `ENABLE_REVIEW_DISADVANTAGES = False`
- `ENABLE_RANKING_SIGNALS = True`
- `ENABLE_CONSTRAINT_SIGNALS = True`
- `SYNTHESIS_REPLY_LENGTH = "medium"`
- `SYNTHESIS_TONE = "helpful"`
- `SYNTHESIS_FALLBACK_ENABLED = True`

## What Frontend Still Needs To Do (Later)

Intentionally not implemented in backend Round 1:

- Render backend `agent_reply` in chat/result UX
- Keep existing frontend fallback message for safety if `agent_reply` is empty

Recommended minimal frontend behavior:

1. If `agent_reply` is non-empty, display it as assistant final message.
2. If `agent_reply` is empty/missing, keep current template-based fallback message.
